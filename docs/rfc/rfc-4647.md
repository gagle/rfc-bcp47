# RFC 4647 — Matching of Language Tags

- **Full title**: Matching of Language Tags
- **URL**: https://www.rfc-editor.org/rfc/rfc4647
- **Status**: Best Current Practice (BCP 47)

## Scope

Defines algorithms for matching language tags against language ranges. Used for content negotiation (e.g., Accept-Language headers). This library implements Extended Filtering (§3.3.2) and Lookup (§3.4).

## Section 2.1 — Basic Language Range

```abnf
language-range = (1*8ALPHA *("-" 1*8alphanum)) / "*"
```

- Same structure as a language tag but no requirement to be well-formed or registered
- Special value `*` matches any tag

## Section 2.2 — Extended Language Range

```abnf
extended-language-range = (1*8ALPHA / "*") *("-" (1*8alphanum / "*"))
```

- Wildcard `*` can appear in any subtag position
- Wildcards in non-first positions are skipped during matching (treated as "match anything at this level")

## Section 3.3 — Filtering

Filtering selects the **set** of all tags that match a language priority list. Each range represents the "least specific tag that is an acceptable match."

### Section 3.3.2 — Extended Filtering Algorithm

Step-by-step:

1. **Split** both the range and the tag into subtag lists by `-`
2. **First subtag**: If the range's first subtag is `*`, it matches any tag's first subtag. Otherwise they must match case-insensitively. If they don't match, the entire tag fails.
3. **Process remaining range subtags** (iterate through range subtags):
   - If range subtag is `*`: skip it, advance to next range subtag
   - If no more tag subtags remain: FAIL
   - If range subtag matches tag subtag (case-insensitive): advance both
   - If tag subtag is a **singleton** (1 character): FAIL (cannot skip past singletons)
   - Otherwise: advance only the tag subtag pointer (skip non-matching, non-singleton tag subtags)
4. **When all range subtags are exhausted**: MATCH succeeds

Key behaviors:
- Wildcard `*` in first position matches everything
- Singletons act as barriers — cannot be skipped
- Non-singleton, non-matching tag subtags are skipped (implicit wildcard)
- Results should be deduplicated
- All matching is case-insensitive

### Extended Filtering Examples

| Range | Tag | Result | Reason |
|-------|-----|--------|--------|
| `de-*-DE` | `de-DE` | match | `*` skipped |
| `de-*-DE` | `de-Latn-DE` | match | `*` skipped, Latn skipped |
| `de-*-DE` | `de-Latn-DE-1996` | match | trailing subtags ignored |
| `de` | `de-x-DE` | no match | singleton `x` blocks |

## Section 3.4 — Lookup Algorithm

Lookup selects a **single** best-matching tag from available tags. Each range represents "the most specific tag that is an acceptable match."

### Algorithm

1. Start with the full language range
2. Try to find an exact match (case-insensitive) against available tags
3. If no match:
   - Remove the rightmost subtag from the range
   - If the new rightmost subtag is a singleton (1 character), also remove it
   - Repeat from step 2
4. If the range is fully exhausted with no match, try the next range in the priority list
5. Return default value if no range matches

### Truncation Rules

- Remove rightmost subtag first
- If truncation leaves a singleton as the new rightmost subtag, that singleton is also removed (never leave a dangling singleton)
- Continue until match found or range exhausted

### Truncation Example

Starting with `zh-Hant-CN-x-private1-private2`:

1. `zh-Hant-CN-x-private1-private2`
2. `zh-Hant-CN-x-private1`
3. `zh-Hant-CN` (singleton `x` removed with `private1`)
4. `zh-Hant`
5. `zh`
6. (default)

### Wildcard Handling

- `*` is skipped in lookup unless it's the only/last range
- If `*` is the only range, return the default value

### Key Properties

- First-match-wins from the priority list
- Case-insensitive matching
- Returns the original casing of the matched tag (not the range's casing) — *implementation choice; RFC 4647 does not explicitly specify which casing to return*
